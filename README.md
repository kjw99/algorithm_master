### 📂 폴더 구조 및 파일 규칙

#### 1️⃣ 폴더 구조
본인 이름의 폴더 하나만 사용하며, 모든 파일은 그 안에 쌓습니다.
```text
root/
└── [이름]/
    ├── W01-1_BOJ_2606.py
    ├── W01-2_BOJ_1000.py
    ├── W01-3_BOJ_2579.py
    └── ...
```

#### 2️⃣ 파일 이름 규칙 (Naming Convention)

주차를 파일명 앞에 붙여 자동 정렬되도록 합니다.

* **형식:** `W[주차]-[순번]_BOJ_문제번호.확장자`
* **예시:** `W01-1_BOJ_2606.py`


#### 3️⃣ 커밋 메시지 규칙

누가 올렸는지는 폴더와 프로필로 확인 가능하므로, **메시지는 최대한 간결하게** 작성합니다.

* **기본 형식:** `타입: 주차-순번 문제번호 메시지`

| 타입 | 의미 | 사용 예시 |
| --- | --- | --- |
| **`feat`** | 문제를 새로 풀어서 올릴 때 | `feat: W01-1 2606 풀이` |
| **`refactor`** | 코드를 수정하거나 개선할 때 | `refactor: W01-1 2606 성능 개선` |
| **`docs`** | README 등 문서 수정 | `docs: 스터디 규칙 업데이트` |
| **`fix`** | 오타나 잘못된 경로 수정 | `fix: W01-2 파일명 수정` |
| **`comment`** | 주석 추가 및 변경 | `comment: W01-1 상세 주석 추가` |
