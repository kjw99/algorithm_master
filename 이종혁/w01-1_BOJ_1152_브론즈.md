# 오늘의 알고리즘 (2026.01.24)

## 오늘 푼 문제
- **백준 1152번**: [단어의 개수](https://www.acmicpc.net/problem/1152) (Bronze 2)

---

## 내 풀이

```python
# 문자열을 입력받아 공백 기준으로 쪼개서 리스트에 담기
# split()은 인자 없이 쓰면 앞뒤 공백을 알아서 쳐내고 단어만 남겨줌
lst = input().split()

# 리스트에 담긴 단어 개수 출력
print(len(lst))
```

## 문제설명
- 영어 대소문자와 공백으로 된 문자열이 주어지는데, 여기서 단어가 총 몇 개인지 세는 문제.
- **주의할 점**: 문자열 앞이랑 뒤에 공백이 올 수 있고, 단어는 공백 하나로 구분된다는 조건이 있음.

## 문제풀이
- **어떻게 풀었나**: 처음에는 `input().split()`을 썼을 때 앞뒤 공백이 어떻게 처리될지 고민했는데, 파이썬 `split()`은 기본적으로 양 끝 공백을 무시하고 실제 단어들만 깔끔하게 리스트로 만들어준다는 걸 활용함.
- **예외 처리**: 만약 입력이 공백 한 칸(`" "`)만 들어오면, `split()` 결과가 빈 리스트 `[]`가 되니까 `len()`을 찍으면 자연스럽게 `0`이 나옴. 따로 `if`문 안 써도 예외 처리가 돼서 편함.
- **성능**: 문자열 길이가 100만이나 되지만, 한 번만 훑고 지나가는 $O(N)$ 방식이라 시간 초과는 걱정 없음.

## 배운점
### 1. split() vs split(' ') 차이점 명확히 알기
- `split()` (인자 없음): 공백이 연속으로 나와도 하나로 보고, 문자열 앞뒤 공백은 싹 다 제거하고 시작함.
- `split(' ')`: 딱 공백 한 칸을 기준으로 다 나눠버려서, 앞뒤에 공백이 있으면 빈 문자열(`''`)까지 리스트에 넣어버림. 이번 문제처럼 단어 개수 셀 때는 그냥 `split()` 쓰는 게 정답인 듯.

### 2. 무지성 map(int, ...) 지양하기
- 숫자가 섞여 있는 입력만 풀다가 습관적으로 `map(int, ...)`를 쓰려 했는데, 문자열 입력일 때는 당연히 `ValueError`가 난다는 걸 다시 상기함. 문제 조건을 보고 데이터 타입부터 확인하는 습관을 들여야겠다.

### 3. 실무 관점의 참견 (데이터 정제)
- 실제 앱 만들 때 유저가 아이디나 비밀번호 입력할 때 실수로 넣은 공백 때문에 로그인 실패 뜨면 짜증 나는데, 이런 `Sanitize`(정제) 로직이 왜 기초이면서 필수인지 알 것 같음. 앞뒤 공백 처리는 선택이 아니라 필수다.
